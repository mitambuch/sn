## Two cases for Option B

1. **B = only different start times** (most likely): just fill the `b`
   values in the existing `options` objects (and add `options` to any extra
   entries that differ). Zero component change. Done.
2. **B = a genuinely different programme** (different activities/bodies, not
   just times): extend the data model — give entries per-option `body`/`label`
   (e.g. `body: string | Record<OptionKey, string>`) OR hold two timelines
   keyed by option, and branch in `Day()`'s render. The badge + marking
   already accommodate it; only the data shape + the one render line change.

## Gotchas (cost time this session — do not rediscover)

- **Commit/PR messages: no literal double quotes** in PowerShell here-strings
  — they break `git commit -m`. Use quote-free text or `git commit -F file`.
  See [[powershell-heredoc-double-quotes]].
- **Local `git merge` to main is denied** by the harness guardrail → always
  ship via PR + `gh pr merge`.
- **Netlify** serves saw-next.ch (Vercel also builds). Deploy is auto on push
  to main, runs `pnpm build` (not `pnpm validate`), so the red bundle-size CI
  check does **not** block deploy.
- `siteConfig.url` = https://saw-next.ch at prod build (set in CI env).
