## Décisions taken

1. **Pattern Vault + pg_net** plutôt qu'Edge Function pour Resend —
   pas de déploiement Deno requis, trigger atomic avec INSERT,
   secret jamais exposé au client. Edge Function réservée pour
   plus tard si besoin de template logic plus riche.
2. **Email opérateur hardcodé** `mitamburini@gmail.com` dans la
   migration 0004 pour MVP — à swap pour `salvatore@sawnext.studio`
   quand domaine acheté + DNS verified dans Resend.
3. **`onboarding@resend.dev` comme sender** pour le test mode
   Resend (sans domaine vérifié, ne peut envoyer qu'à l'email du
   compte Resend = celui de l'owner, ce qui suffit pour le MVP).
4. **DEV session prend la priorité même si hasSupabase=true** —
   pour que tests + override manuel marchent indépendamment du
   .env.local. AuthContext useEffect skip si readDevSession()
   non-null.
5. **signIn hydrate state synchronously** avant retour — sinon
   navigate(/account) lose la race contre onAuthStateChange et
   RequireAuth kick back to /login.
6. **vitest mock global de @/lib/supabase** — évite que les tests
   pickent .env.local et tentent des calls live.
