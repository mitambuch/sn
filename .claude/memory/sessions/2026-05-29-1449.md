### SeoHead wire (resolveField chain)

[src/components/features/SeoHead.tsx](../../../src/components/features/SeoHead.tsx)
résout maintenant :

- `description` : prop > Sanity siteConfig.seoDescription[locale] > local
- `brandName` : Sanity siteConfig.siteName[locale] > local siteConfig.name
- `baseTitle` : Sanity siteConfig.seoTitle[locale] > brandName
- `fullTitle` : prop title + " | " + brandName si fournie
- `ogImage` : prop > local (Sanity image asset URL resolution = future)

Limitation honnête : `scripts/prerender-meta.js` (postbuild) émet les
valeurs locales au build time. Les bots qui exécutent JS (Google,
Twitter modernes) voient les valeurs Sanity ~100ms après hydration.
Bots non-JS = local defaults. À fixer en v1.3.x avec un Sanity fetch
build-time.

   v1.3.x quand UI sera définie.

2. **Migration 0014 grants `sb_secret_*`** — considérée mais skipped.
   Le nom du role mappé par les nouvelles API keys Supabase est
   incertain sans accès au dashboard. Écrire une migration au
   blind = risque casser plus que fixer. La friction

   et [session 1406](2026-05-29-1406.md) documentent le workaround
   SQL Editor qui marche aujourd'hui.

3. **Prerender Sanity fetch** — nécessite refactor build pipeline
   (vite plugin custom ou postbuild script async). Pas surgical, à
