### Phase 1 — Depth tokens + Card atom

**`src/index.css`** : 4 nouveaux tokens dans `@theme`, auto-mappés
Tailwind 4 en utilités :

```css
--radius-card: 1.25rem;          /* 20px — assumé HNW */
--radius-media: 0.875rem;        /* 14px — radius interne */
--shadow-card-rest: inset 0 1px 0 0 fg/8%;  /* inner top specular */
--shadow-card-hover:
  0 8px 24px -12px rgb(0 0 0 / 0.55),
  inset 0 1px 0 0 fg/18%;
```

**`src/components/ui/Card.tsx`** : réécrit Apple-closed.
- Container : `rounded-card border border-border bg-surface

- Nouveau : prop `to` rend `<Link>` (SPA), prop `href` rend `<a>`

## Décisions taken

1. **`Card` atom UN seul, composition par slots** (vs N variantes
   typées). Card.Media + Card.Body + slots discrets — pattern Apple
   App Store / Linear. Backward-compat children-only préservée pour
   playground + stats.
2. **Tokens dans @theme block**, pas dans `:root` — Tailwind 4 auto-
   mapping. Évite les `[var(--radius-card)]` bracket syntax (warning
   eslint canonical-classes).
3. **Dual-shadow consolidé en un seul token** `--shadow-card-hover`
   contenant drop + inner highlight intensifié. Permet le hover en
   1 utilité canonical (`hover:shadow-card-hover`) au lieu de
   bracket comma-separated.
4. **Aspect-ratio pragmatique** (pas uniformément 4:3). Property
   3:4 et Timepiece/Artwork 1:1 sont des conventions du segment —
   les forcer en 4:3 aurait cassé la lecture domain. La cohérence
   vient du WRAPPER, pas du contenu image.
5. **Prop `to` ajoutée à Card** pour Link SPA, en plus de `href`
   plain. Évite la régression de full-page-reload sur les tiles
   admin (qui utilisaient `<Link>` avant).
6. **Tokens light-mode override séparés**. Drop shadow ramenée à
   12% (vs 55% dark) pour cohérence visuelle cross-mode sans casser
   l'effet "carved-in" propre au dark.
7. **Date badge Event en Card.Overlay** plutôt que bloc externe
   à côté du meta — gain de signature visuelle, frosted glass
   premium, position top-left ne masque pas le sujet image.

## Frictions

1. **`exactOptionalPropertyTypes: true`** force `prop?: T | undefined`
   partout sur les props optionnels. 9 interfaces à mettre à jour
   dans Card.tsx avant que les wrappers ne typecheck. Identifié au
   1er validate, 2 min de fix.
2. **Tailwind 4 canonical classes** : `rounded-[var(--radius-card)]`
   warning suggestCanonicalClasses → forme `rounded-card`. Il a fallu
   restructurer les tokens (consolider `--highlight-card-top` dans
   `--shadow-card-rest`) pour que les hover dual-shadow passent en
   canonical.
3. **Commit-msg hook** : title >72 chars rejected une fois. Reformulé
   "feat(ui): Apple-closed Card system for 7 domain cards + admin".
   Warning "no WHY line" passé (le hook cherche "WHY:" avec colon
   probablement, mon body avait "WHY" en section header — accepté
   avec warning, commit landed).
4. **Lint-staged stash inutile** : 2 round-trips parce que husky a
   re-stash et re-apply après chaque tentative commit. Pas grave,
   process automatique.
