# Session — Dates d'événement flexibles (après-midi)

## Contexte

Retour client : sur les événements, certaines offres sont dispo toute
l'année ; le schéma forçait début + fin de date. Demande d'une case
« toute l'année » ou d'un champ libre.

## Ce qui a été fait

Branche `feat/event-flexible-dates`, commit `bec9140`.

Sélecteur `dateMode` (exact / allYear / free) sur l'event Sanity — couvre
les deux demandes client via un seul mécanisme. Détail de la décision :
`decisions/2026-06-01-event-flexible-dates.md`.

Fichiers : schéma `studio/schemas/documents/event.ts` (dateMode +
startsAt/endsAt conditionnels + dateLabel + preview), `types/event.ts`
(startsAt optionnel), `lib/sanityQueries.ts` (×2 projections), nouveau
helper `features/events/eventDate.ts` + son test (6 cas), `EventCard`,
`EventDetail`, `SharePage`, i18n FR/EN (`events.allYear`),
`scripts/validate-sanity-schema.js` (dateMode → ALLOWED_RAW).

## Validation

`pnpm validate` vert sauf build (garde `VITE_APP_URL` volontaire — connue).
Build confirmé OK avec URL factice. Helper testé unitairement.
