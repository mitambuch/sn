## Diagnostic + fix (4 points)

1. **Contenu tronqué** — `PUBLIC_FICHE_PROJECTION` (sanityQueries) ne récupérait
   ni `programme` ni `capabilities` ; `PublicFichePanel` ne rendait que héro/
   titre/résumé/description/galerie. → projection + type + rendu d'une section
   **Programme** (events = heure+titre+détail, concierge = titre+détail) + d'une
   liste **Ce qui est inclus** (capabilities concierge). Prix/specsheet restent
   volontairement derrière l'accès (discrétion HNW) — signalé à l'owner.
2. **CTA « Manifester mon intérêt » absent sur certaines fiches** — sur une
   fiche valide le CTA est **inconditionnel**. La vraie variable est le
   back-office : un doc non taggé `visibility: public` (ou sans image) n'apparaît
   pas comme fiche publique → pas de bouton. Code rendu cohérent ; cause
   visibility à confirmer par l'owner sur les fiches précises.
3. **CTA → tunnel d'accès au lieu d'un contact** — nouveau
   `ExperienceInterestModal` (contact simple → `access_requests`, anon-insert +
   email opérateur, zéro migration). Câblé sur `PublicFichePanel` + `SharePage`.
   Tunnel d'accès conservé pour les vrais CTA « devenir membre ». Décision :
   [[public-experience-interest-contact-form]].
4. **Espagnol incomplet** — code OK : la gate passe `$locale=es`,
   `GROQ_PUBLIC_FICHE` a les branches es, `es.json` a la parité `share.*`. Le
   FR affiché en ES = **trou de contenu Sanity** (Jaquet Droz et les
   `conciergeService.programme` créés après le miroir es du 2026-06-09 n'ont pas
   de `*.es`) → fallback FR cohérent (par design). Durcissements code : `localeStr`
   gagne une branche `es` (+ arg `locale`), la nouvelle projection programme est
   es-aware. Le 1 hardcode FR restant (`SharePage` lede) → `t('share.interest.lede')`.
