## What landed sur main

### Bloc 1 — Landing chrome (sélecteur de langue)

**Commit `7bdc89e`** → merge `a6f32c1`

- [src/pages/Home.tsx](../../../src/pages/Home.tsx)
- Cause : la Home `/:locale/` est mountée **outside `PublicLayout`**
  (full-bleed volontaire, TerminalBar agit comme chrome) → le
  `<Header />` public avec son `LanguageSwitcher` n'est jamais
  monté.
- Fix : `useLocale()` plug dans `HomeContent`, passe
  `currentLocale + setLocale` au sous-composant local
  `TopCornerChrome`, ajoute un mini groupe `fr / en` inline à
  gauche du bouton `INDEX`.
- Style : font-mono uppercase tracking-widest, `text-white` +
  `mix-blend-mode: difference` hérité du parent → auto-inverse sur
  n'importe quel fond (hero vidéo, marquee clair, bg-ink).
- Active opacity-100, inactive opacity-50 hover:opacity-100, ARIA
  `role="group"` + `aria-pressed`.
- Décision passée à respecter (2026-05-14) : le `AuthHeader` sur
  `/account/*` + `/admin/*` reste sans switcher — c'est volontaire.

### Bloc 2 — Cleanup déploiement (3 commits, 1 merge `cb537c6`)

  paths publics) — vérifié via `WebFetch saw-next.ch/sitemap.xml`
  après redéploiement Vercel.

**`cac32a1` — `fix(ci)` VITE_APP_URL injecté step Validate**
- Cause CI rouge depuis 27/05 (4 runs successives) :
  `vite.config.ts` exige `VITE_APP_URL` en prod build pour ne pas
  fallback canonical sur localhost. GitHub Actions n'avait pas la
  var → `pnpm validate` crash au step `build`.
- Fix initial : `env: VITE_APP_URL: https://saw-next.ch` au step
  Validate (inline, pas secret — URL publique).
