## Ce qui a shippé

### `0d1aaa6` — 6 types de menus + showcase section

6 composants autonomes dans `src/workbench/playground/menus/` :

1. **BrutalistNav** — wordmark oversized + divider b-2 brut
2. **EditorialHeader** — masthead centré + micro-nav avec slashes

Plus :
- `shared/MenuFrame.tsx` — wrapper chaque preview (name + ethos + path copiable)
- `sections/MenusSection.tsx` — assemble les 6 en section "00" du playground
- Wire-up dans `src/pages/Playground.tsx`, `sections/index.ts`, `shared/index.ts`

Contraintes respectées :
- Tokens only (`bg-bg`, `text-fg`, `text-accent`, etc.) — zéro hex
  hardcodé.
- Responsive : `sm:` / `md:` / `lg:` couvrent 320 → 1920 px **inclus
  portrait monitors** (~1080 px width).
- Touch targets 44×44 mini sur mobile.
- Accessibilité : `aria-label`, `aria-expanded`, `role="dialog"`,
  Escape ferme les overlays.

**Validation** : `pnpm validate:fast` vert (lint + typecheck app +
typecheck studio). **QA visuel non-faisable depuis Claude** — à
vérifier par l'owner via `pnpm dev` + `/playground`.

## Méta-observations

### Dispatch non-utilisé
Aurais-je pu dispatcher les 6 menus à worker-sonnet en parallèle ? Non
— chaque menu est une décision **design** (R2), pas un pattern mécanique
(R1). Dispatch aurait produit 6 menus similaires, pas 6 identités
distinctes. Main-thread était correct.

### Intent routing non-déclenché
Aucune intention "création de X" naturelle cette session — l'owner a
posé un plan stratégique, pas une ask qui matchait une entry du tableau.
Normal, pas une lacune.
