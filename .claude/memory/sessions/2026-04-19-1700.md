## Tech debt noté (pas fixé aujourd'hui)

**Header.tsx duplique LocaleProvider.setLocale**. ChatGPT R2 a
réimplémenté `localeFromPath()` + `stripLocale()` + navigation manuelle
au lieu d'appeler `useLocale()` du contexte existant. Ça marche (Header
est rendu dans LocaleProvider via RootLayout), mais c'est DRY-violant.

**Bonus** : ChatGPT preserve `search+hash` lors du switch locale, alors
que `LocaleProvider.setLocale` actuel perd search+hash. Bug
préexistant dans LocaleProvider ligne 56-65.

**Action future** (P2, post-release) : refactor Header pour utiliser
`useLocale()`, fix LocaleProvider pour preserve search+hash. Branche
dédiée, ~15 min.

## Lecture du désaccord avec ChatGPT

ChatGPT R2 prétendait que les IDs modèles Claude 4.7/4.6/4.5 n'existent
pas ("je vois claude-opus-4-1-20250805 etc"). **Faux dans ce contexte**
— mon system prompt Claude Code actuel liste explicitement Opus 4.7,
Sonnet 4.6, Haiku 4.5 comme IDs valides. ChatGPT a consulté des docs
Anthropic publiques obsolètes ou cache.

**Leçon** : le pattern cross-audit LLM ne veut pas dire "accepter tout
ce que le LLM externe dit". Distinguer :
- **Bugs factuels vérifiables** (token count, filename sort, doc
  reference order) → action immédiate
- **Opinions structurelles** (token diet, dispatch LARP) → débattre,
  peser
- **Claims sur IDs / docs externes** → valider contre la source (pour
  les modèles, mon system prompt ; pour les libs, docs officielles)
