## Ce qui a livré

### Bloc unique — Salva autonomy P0

1. **Studio Document Action "Partager cette fiche…"** sur les 7 types
   de docs (event/property/timepiece/artwork/journey/conciergeService/
   article). Le dialog s'adapte à la visibility :
   - `private` → explique le verrou et comment l'ouvrir
   - `public` → URL canonique + bouton WhatsApp + Email + Copier
   - `shareCode` → bouton "Ouvrir le générateur" qui deeplink sur
     `/admin/share-codes?docType=event&docId=evt-01` form pré-rempli
2. **studio/components/ShareDialog.tsx** : UI body en @sanity/ui
   (Box/Button/Card/Code/Stack/Inline/Text) — découplé du bundle app.
3. **src/lib/sharing.ts** : helpers purs sans dépendance DOM —
   `buildShareMessage` (FR/EN brand voice), `buildWhatsAppUrl`,
   `buildMailtoUrl`, `buildSmsUrl`, `copyToClipboard` (avec fallback
   textarea pour browsers anciens).
4. **src/components/ui/ShareActionRow.tsx** : atom UI réutilisable —
   row de 4 boutons (WhatsApp / Email / SMS / Copy). Variant "row" ou
   "compact". WhatsApp en primary (canal HNW privilégié).
5. **src/pages/SharePage.tsx rewrite** : quand status=valid, fetch via
   `GROQ_SHARED_FICHE(type, id)` + `useSanityItem`. Render premium :
   hero image 16/9, titre, summary, description, share row pour que
   le destinataire puisse re-partager. Fallback placeholder propre
   quand Sanity offline/vide. OG metadata injecté via React 19 head
   hoisting (title + meta description + og:image + noindex).
6. **src/pages/AdminShareCodes.tsx upgrade** :
   - Lecture `?docType=X&docId=Y` via `useSearchParams` (initial form
     state, pas de useEffect-set-state — règle React 19 stricte).
   - Action **Révoquer** par ligne (UPDATE status='revoked' + confirm
     window + toast + optimistic UI).
   - Lien **WhatsApp** par ligne (deeplink prêt avec brand message).
   - Action **Copier URL** par ligne.
   - Status revoked en `line-through` pour visibilité.
7. **studio/schemas/_helpers/visibilityField.ts** : description du
   field complétée avec hint "menu ⋮ → Partager cette fiche…" pour
   que Salva découvre l'action depuis le field lui-même.
8. **studio/package.json** : ajout `@sanity/icons` + `@sanity/ui` en
   deps directes (étaient transitives via `sanity`).

## Décisions notables

- **Document Action plutôt que Sanity Tool dédié** — l'action démarre
  depuis la fiche (contexte déjà chargé) au lieu d'un menu séparé. Un
  Tool dupliquerait AdminShareCodes côté Studio.
- **Pas de génération de code depuis Studio directement** — Studio
  n'a pas la session Supabase admin de Salva. Le deeplink vers
  `/admin/share-codes?docType=X&docId=Y` est plus simple et plus sûr
  (auth déjà gérée frontend).
- **ShareDialog standalone** (n'importe pas depuis src/) — le bundle
  Studio reste découplé du bundle app. La duplication de
  `buildShareMessage` est minime et stable (brand voice).
- **Channels par défaut** : WhatsApp / Email / SMS / Copy. WhatsApp en
  primary parce que c'est le canal owner explicit ("envoie sur
  WhatsApp"). SMS pour ceux qui n'ont pas WhatsApp.
- **OG metadata sur /share/:code = noindex** — privé par nature,
  pas d'indexation Google. Mais OG title + image quand même pour le
  rich preview WhatsApp/iMessage.
- **Pré-existing tech debts pas fixés** : raw `string` fields dans
  schemas (event/property/timepiece), 2 FR strings dans AdminShareCodes
  (placeholder + empty label). Out-of-scope, à fixer en follow-up
  ciblé.
