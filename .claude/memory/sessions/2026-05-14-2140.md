## Key decisions logged

### Sanity vs /admin separation (R2 architectural)

Question : "should EVERYTHING live in Sanity, including codes ?"
Verdict : no — Sanity for CONTENT (texts, photos, fiches catalogue),
Supabase for STATE (codes, inquiries, users, sessions). 3 reasons :
- Sanity has no Row Level Security (HNW confidentiality breach risk
  if a 2nd editor ever gets access)
- Sanity has no atomic transactions (the FOR UPDATE lock for
  single-use redemption can't be reproduced)
- Auth for site members lives in Supabase by construction

Decision recorded in `decisions/2026-05-14-admin-architecture-split.md`
(to write next session).

### /admin/catalogue as monitoring hub, not authoring tool

Owner showed the brutal HTML form for "Add a watch" — verdict : don't
fix it, REMOVE it. Sanity Studio already does drag-drop + hotspot +
mobile UX 10× better. /admin/catalogue is now a read-only grid that
deep-links every row into the matching Sanity Studio doc.
