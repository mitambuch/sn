## Frictions rencontrées

### 1. Owner non-promu admin sur prod Supabase (toujours)

Source du symptôme "/admin rien ne joue". RLS bloque tous les
operator queries pour un profil non-admin. Le SQL UPDATE doit être
run par owner (le `sb_secret_*` n'a pas le grant sur la colonne
`role`, friction documentée en session 1406).

### 2. Pipe validate qui plante sur `memory:index:check`

`pnpm validate` (gates → memory:index:check) a fail après écriture
du friction Swatch parce que l'INDEX n'avait pas été régénéré.
Régen via `pnpm memory:index` → retry validate → ✓. Pattern : tout
ajout dans `.claude/memory/{decisions,feedback,patterns,frictions,sessions}/`
doit être suivi de `pnpm memory:index` avant `pnpm validate`. À
intégrer dans un hook pre-commit (futur).

1105 confirme Studio branché en prod, contenu possiblement édité
