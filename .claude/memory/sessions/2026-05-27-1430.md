## Friction encountered

- **`react-hooks/set-state-in-effect` lint** rejected the initial
  Interlocutor refactor where I had `useEffect` syncing focalKey to
  focalDefaultKey on member list change. Fixed by deriving
  `effectiveFocalKey` during render instead. Cleaner anyway.
- **`useAuth must be used inside <AuthProvider>`** broke 3
  AccountCatalogue tests after wiring useSavedItems to AuthContext.
  Fixed by reading userId directly from `supabase.auth.getSession()`
  + `onAuthStateChange` listener — decouples the hook from the
  AuthContext entirely. Tests pass with zero modification.
- **`pnpm validate` build step fails** without `VITE_APP_URL` set
  locally. Documented in DEMO-CHECKLIST.md — vite.config.ts has a
  prod-build guard that throws if neither VITE_APP_URL nor
  VERCEL_URL is set. Local validate works with
  `VITE_APP_URL=http://localhost:5173 pnpm build` ; deploy works
  because Vercel injects VERCEL_URL.

## Patterns reused

- **Sanity hook pattern** (useTeamMembers) mirrors useAdminCatalogue +
  useLandingContent : `{ data, loading, error, usingFallback }`
  shape, `cancelled` flag for unmount safety, fallback semantics
  when `hasSanity === false` or dataset empty
- **Optimistic-local + background-sync** pattern (useSavedItems) is
  the recipe documented in `SHIP-AUDIT.md` (2026-05-14 draft) — now
  shipped
- **Auth-decoupled hook** : reading `supabase.auth` directly instead
  of `useAuth` keeps test surface clean. Future hooks doing
  user-scoped DB writes should follow this pattern
