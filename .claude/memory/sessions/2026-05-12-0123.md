## Décisions taken

1. **Card atom est le système** — 7 cards domain + admin + saved +
   inquiries listings + wizard category buttons + sidebar tiles
   utilisent tous Card.* slots. 1 source de vérité pour radius/
   surface/border/hover/focus.
2. **Tokens depth dans @theme** — Tailwind 4 auto-utility-gen
   (`rounded-card`, `shadow-card-rest`, etc.) au lieu de
   `rounded-[var(--radius-card)]` bracket. Canonical, eslint-clean.
3. **Badge top-left = TEMPORAL only** — Event date, Journey duration,
   Article publication date. PAS sur produits (Property/Timepiece/
   Artwork) où l'image est le signal.
4. **PriceBlock = produits only** — Property/Timepiece/Artwork ont
   PriceBlock (avec PriceTag pill premium). Event/Journey/Concierge/
   Article = pas de price slot (différenciation sémantique par
   absence).
5. **Mobile drawer pattern AppLayout** — sidebar fixed top-0 bottom-0
   avec border-r continu, hamburger top-left z-50 md:hidden, drawer
   slide-in avec backdrop. Body scroll lock, Escape close, route
   change auto-close (derived state pour éviter useEffect+setState
   rule).
6. **Wizard adaptive par category** — switch case sur la step Details
   pour rendre les bons fields. Tous optional, callback fast lane
   en review. 5 catégories structurées + 1 free-form (other) avec
   ton chaleureux différencié.
7. **priceCHF? sur les types** — déjà présent sur Timepiece,
   probablement à étendre sur Property/Artwork/Event/Journey pour
   mapping Sanity facultatif. Sur le hero TimepieceDetail, PriceTag
   surface la valeur si renseignée, fallback "Sur demande" sinon.

## Frictions

1. **eslint react-hooks/set-state-in-effect** — `setDrawerOpen(false)`
   dans useEffect sur pathname change refusé. Fix : derived state
   pattern (compare prevPath inline). Légèrement non-idiomatique mais
   valide.
2. **Commit-msg hook** — title >72 chars refusé plusieurs fois. Fix :
   raccourcir titres. Aussi : `footer-leading-blank` warning sur les
   commits où la HEREDOC newline manque avant Co-Authored-By, mais
   commit landed quand même (warning non-bloquant).
3. **Tailwind 4 canonical class warnings** — `[&>*]:pointer-events-
   auto` → `*:pointer-events-auto`, `rounded-[var(--radius-card)]` →
   `rounded-card`, `shadow-[var(--highlight-card-top)]` →
   `shadow-(--highlight-card-top)`. Restructure tokens pour passer
   canonical (consolider `--shadow-card-rest` au lieu de
   `--highlight-card-top` + bracket).
4. **i18n key missing** — `account.greeting.morning` n'existait pas
   (affichait la clé brute). Fix : ajouter 3 clés. Aussi
   `timepieces.specifications` manquait, drop le label inutile.
5. **Card.atom onClick a11y** — Card ne supporte pas onClick comme
   button (div onClick = pas keyboard accessible). Workaround :
   button styled comme Card pour PersonalisedRequestSection CTA.
   Pour la suite : extend Card avec `as="button"` + onClick.
6. **TypeScript exactOptionalPropertyTypes** — props avec `?: T`
   forcent `T | undefined` partout. 9 interfaces à mettre à jour
   dans Card.tsx (mineur mais visible).
7. **Owner correctness loop** — owner a corrigé 4 fois en série
   sur les mêmes thèmes (boutons wrap, padding mobile, etc.). La
   règle "mobile-first restructure" a été tâtonnée 3 fois avant
   d'être capturée correctement. Friction d'apprentissage : utiliser
   les patterns ANCRÉS dans memory dès qu'ils sont validés.
