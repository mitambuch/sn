## Frictions

1. **`pnpm setup` collide avec built-in pnpm command** — il faut
   `pnpm run setup` pour atteindre le script du package. Le wizard
   bascule en mode `--yes` si stdin non-TTY → utilise dirname comme
   slug (`sn`). Patch manuel ensuite a fonctionné.
2. **`@types/*` alias** — réservé par TypeScript pour
   DefinitelyTyped. TS6137 sur `import from '@types/auth'`. Switch
   sur `@/types/auth` (alias `@/*` existant).
3. **react-refresh/only-export-components** — exporter useAuth +
   AuthProvider dans le même fichier déclenche le warning. ThemeContext
   passe sans warning malgré le même pattern, racine non identifiée
   (probablement version eslint-plugin différente). Workaround :
   `// eslint-disable-next-line` sur useAuth, HMR cost négligeable.
4. **`navigate()` en RR7 retourne `void | Promise<void>`** —
   no-floating-promises rule fail. Fix : `void navigate(...)`.
5. **`role` prop colide avec ARIA `role`** — jsx-a11y/aria-role
   flag `<RequireRole role="admin">` comme rôle ARIA invalide.
   Renommé en `requiredRole`.
6. **Coverage threshold ratchet** — nouveaux fichiers (layouts,
   guards) ont chuté la couverture sous le seuil. Ajout de 4 test
   files (PublicLayout, AppLayout, AdminLayout, RequireAuth déjà fait
   par worker) pour récupérer.
7. **Tests i18n labels** — locale par défaut = FR donc tests doivent
   matcher "Événements", "Se déconnecter" etc., pas les versions EN.
