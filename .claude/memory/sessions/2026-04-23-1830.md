   MSYS_NO_PATHCONV=1`. À ajouter à la memory frictions si récurrence.
3. **Python /tmp/ path** invisible depuis Python même quand Git Bash
   voit les fichiers. Workaround : copier dans `.claude/memory/
   creative-library/.uipro-temp/` pendant l'extraction, supprimer après.

## Décisions taken

1. **Extraction curée uipro > install plugin**. Risque d'auto-
   activation et de génération de design-systems "safe AI slop" trop
   élevé. L'owner a confirmé : "pas installer le plugin".
2. **Creative-library séparé de memory standard**. Pas un décision,
   un pattern découvert : `memory-index.js` ne scanne pas
   `.claude/memory/creative-library/` (seulement decisions/feedback/
   patterns/frictions/sessions/). Le catalogue reste à 82 entries
   dans INDEX.md, les 22 fiches creative-library ne sont pas indexées
   là-dedans. C'est voulu : creative-library est un **pool de
   ressources**, pas un journal chronologique.
3. **Deps 2026 deferred**. Install en Phase 4 quand un pattern concret
   les consomme, pour éviter bundle weight mort.
4. **Always-loaded rules : +2 (vision-first + creative-ambition)**
   → total 10 always-loaded. Token budget impact estimé ~+3 KB.
   Acceptable vu la discipline value.

## Ce qui va changer dans mes prochains comportements

1. Toute requête UI non-triviale (nouvelle page, section refonte,
   token rework) → je grep `creative-library/mechanics/` avant de
   proposer.
2. Mon bloc VISION contient **toujours** `Mechanics:` avec ≥ 2 slugs
   depuis le catalogue. Sinon c'est `mechanics shallow` → je retravaille.
3. Face à un absolu owner (*"zéro X"*, *"tout Y"*, etc.) → CHALLENGE
   auto (Phase 0 hygiene).
4. Face à un `client.md` placeholder → refus de contenu fabriqué
   (Phase 0 HARD GATE).
5. Réflexe par défaut pour du visuel : r3f + scroll-driven + generative
   bg **avant** carousel + fade. Si je retombe dans le basique,
   warning hook prévu en Phase 5 (session future).
