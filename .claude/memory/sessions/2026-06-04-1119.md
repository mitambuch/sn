# Session — Refonte du tunnel d'accès (mot de passe, un seul flux)

Branche `feat/auth-password-tunnel`, commit `2d2885f`. **Non mergé** — en
attente de tests live owner + réglages dashboard Supabase.

## Contexte

L'owner : le tunnel de connexion est « brouillon », il ne le comprend pas
lui-même. A demandé un tunnel « PUTAIN simple » et « blindé à 100% ». Diagnostic
avant code (R2) → 3 défauts réels identifiés, pas un ressenti.

## Diagnostic (3 bugs, vraies causes)

1. Email d'acceptation pointe vers `sawnext.ch` (faux) au lieu de `saw-next.ch`
   (Resend + CI). → tombe sur Vercel. Owner a tranché le domaine canonique.
2. Aucun système de mot de passe : login 100 % lien magique, mode mot de passe
   mort, double-email + double-code à l'onboarding.
3. Deux UIs de login divergentes (page `Login.tsx` + `LoginModal.tsx`).

## Livré

- **AuthContext** : `registerWithCode` (verify → signUp password → redeem),
  `requestPasswordReset`, `updatePassword`. Retiré magic-link/redeem/confirm.
  Tous les redirects email via `env.APP_URL`, jamais `window.location.origin`.
- **LoginPanel** (nouveau) : UI + logique partagées. `Login.tsx` (page) et
  `LoginModal.tsx` (overlay) sont des coques fines → plus de divergence
  possible. Voir pattern [[shared-panel-two-shells]].
- **ResetPassword** + route `/reset-password` + `ROUTES.RESET_PASSWORD`.
- **Onboarding** réduit à un écran de bienvenue (code déjà consommé).
- **AccessRequestModal** « J'ai un code » → `/login` première connexion,
  pré-rempli ; accepte le code brut 8 car de l'email.
- **Migration 0021** (toujours pending) : `sawnext.ch` → `saw-next.ch` + copie
  clarifiée.
- **vitest** : `testTimeout` 5s → 15s (flake gate, voir
  [[vitest-coverage-testtimeout-flake]]).
- Tests : AuthContext mis à jour, LoginPanel (toutes branches), ResetPassword.
