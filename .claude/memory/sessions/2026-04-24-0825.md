## Frictions

1. **SessionStart gitStatus périmé** — snapshot au démarrage montrait
   l'ancienne branche + untracked files, alors que l'owner avait
   merged/pullé entre-temps. M'a conduit à un diag initial partiellement
   faux (j'ai recommandé "release v6.9.0" qui était déjà cut hier).
   Leçon : **toujours relancer `git log --oneline -15 && git branch
   --show-current` en DÉBUT de turn audit**, ne pas trust la snapshot
   SessionStart passé quelques minutes.

2. **Flake chromium a11y** — `e2e/a11y.spec.ts:28:3 Home (/)` fail en
   run complet, pass en run isolé. Pattern connu de contention
   Playwright quand chromium+firefox+webkit tournent simultanément
   sur Windows. Pas une régression. À surveiller — si récurrent,
   ajuster `playwright.config` pour sérialiser a11y specs.

## Décisions taken

1. **Override uuid plutôt que dismiss** — le précédent `pnpm.overrides`
   établit clairement le pattern. Test empirique (`lhci healthcheck`)
   plus fiable que spéculation. Override coûte 1 ligne, clean les
   alerts remote, zéro régression mesurée.
2. **Pas de release v6.9.1 immédiate** — voir RELEASE CHECK. Les 3
   commits chore(memory) + 1 fix(deps) depuis v6.9.0 ne justifient
   pas de cut. Ship sur le prochain feat.
