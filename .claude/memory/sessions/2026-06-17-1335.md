# Session — Split /QR + /presentation, redesign dans l'univers landing

Suite de [[session-2026-06-17-1315]]. La page salon livrée le matin était
**« trop basique »** (feedback owner). Refonte + split.

## Fait (mergé main `94232a3`, déployé)

- **Split en deux pages** :
  - `/QR` (`QRPage.tsx`) = porteur de QR habillé (grain, fond gris, marquee ✦,
    chrome mix-blend) ; le QR pointe vers `/presentation`.
  - `/presentation` (`Presentation.tsx`) = la présentation riche.
  - Flow salon : client ouvre /QR → montre l'écran → scan → /presentation.
- **/presentation reconstruite dans le langage de la landing** : hero section
  **ink** + grain + mot cyclant **mix-blend-difference** + bande méta terminal
  (GPS hairline + 3 meta-rows), **Marquees ✦**, **rythme light/ink** (plateforme
  en section noire), SectionTag ↘, hairlines, reveals ease-luxe. Réutilise
  Marquee/SectionTag/Reveal/useCyclingWord/BrandMark (une page peut importer les
  features landing ; le form extrait dans `features/presentation/` pour la
  boundary).
- L'ancienne `QRPresentation.tsx` supprimée. QR overlay retiré → clés
  `qr.qrcode` + `qr.cta.shared/shareText` nettoyées. Nouvelles clés
  `qr.hero.{gps,m1-3}` + `qr.qrpage.*` (parité 64/64/64).

## Leçon (feedback écrit)

[[2026-06-17-presentation-not-basic]] — récidive de la règle « push luxe »
([[2026-05-11-luxury-push-further-bias]]). Une page de ce projet **doit**
réutiliser grain + rythme ink + marquees + SectionTag, sinon elle est plate.
Checkpoint screenshot avant merge (fait : mobile/desktop des deux pages).
