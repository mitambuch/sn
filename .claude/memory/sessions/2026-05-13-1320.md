## Décisions notables

- **8 chars pour share code** (validé owner) — même format que
  invitation_codes pour cohérence visuelle, distinction par table
  + route.
- **Visibility tri-state explicite** plutôt que toggle on/off —
  donne contrôle granulaire à Salva (private = espace membre,
  public = canonical SEO, shareCode = link unique).
- **Sanity Studio gratuit sanity.studio hosting** (validé owner)
  plutôt que custom domain — réversible plus tard.
- **Inspiration patterns étoiles-aux-atomes** : event.ts donné via
  gh api (raw URLs requièrent token GitHub auth). Pattern adopté :
  groupes par tabs Studio + descriptions explicites éditeur +
  emoji icons.
- **Wiring textes landing → Sanity SKIPPÉ** : trop invasif,
  Salva ne touchera pas ces textes brand. Décision pragmatique
  pour boucler l'essentiel (catalogue + share).
- **Document Action Studio "Générer code" SKIPPÉ** : complexe à
  câbler depuis le Studio vers Supabase. Admin frontend page
  /admin/share-codes plus simple et cohérent avec l'existant.
