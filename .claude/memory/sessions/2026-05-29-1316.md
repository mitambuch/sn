## Friction rencontrée

### Gap bilingue catalogue (P0 audit, non-corrigé)

Voir [frictions/2026-05-29-bilingual-detail-gap.md](../frictions/2026-05-29-bilingual-detail-gap.md).

En creusant : les detail queries GROQ ne ramènent que `.fr`. Les
list queries ramènent `titleEn` MAIS ce champ n'est consommé nulle
part — code mort. Les types TS sont mono-string. Bref, **le
catalogue est mono-FR en pratique**, pas seulement les details.
Pas un bug isolé, un refactor bilingue complet (~2-3h, types +
GROQ + transforms + render).

**Décision owner** : "stop, juste doc memory". Refactor reporté
à v0.4.x/v0.5.0. Friction documentée avec plan d'attaque.

### Coupure de branche parallèle

À 13:18 environ, un check git a révélé que la branche active était
devenue `chore/strip-title-punctuation` (working tree clean, mes
edits dans le stash). Owner clarifie : deux sessions en parallèle,
celle-ci doit rester sur Sanity. Rebascule sur
`fix/sanity-wiring-clean`, stash pop, restauration des fichiers
qui appartenaient à la branche parallèle (`src/locales/fr.json`,
`src/locales/en.json`, `src/features/landing/Interlocutor.tsx` —
tous des strip de ponctuation), commit Footer propre.

Lesson : en parallel multi-session, **toujours vérifier
`git branch --show-current` avant chaque commit**, surtout après
une opération bloquante (hook refusé, pause owner). Le risque de
mélanger les territoires monte avec le nombre de sessions
concurrentes.
